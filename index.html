Git 시작 가이드

Git은 파일의 변경 이력을 안전하게 관리하고, 여러 사람이 동시에 협업할 수 있게 해주는 분산 버전 관리 시스템입니다. 이 페이지에서 Git의 개념, 설치 방법, 핵심 명령어, 실무형 작업 흐름까지 빠르게 익힐 수 있습니다.

Git이 필요한 이유

실수로 덮어쓴 코드 복구, 변경 이력 추적, 기능 단위 브랜치 개발, 코드 리뷰 기반 협업을 쉽게 할 수 있습니다.

핵심 개념 3가지

Repository Commit Branch

저장소, 스냅샷(커밋), 작업 가지(브랜치)를 이해하면 Git의 80%는 끝입니다.

GitHub와의 관계

Git은 버전 관리 도구이고, GitHub는 Git 저장소를 원격으로 호스팅하고 협업 기능(PR, 이슈 등)을 제공하는 서비스입니다.

1) Git 설치 방법

운영체제별로 아래 링크 또는 패키지 매니저를 통해 설치합니다.

운영체제 설치 방법 설치 확인
Windows Git 공식 설치 파일로 설치 git --version
macOS brew install git 또는 Xcode Command Line Tools 설치 git --version
Linux (Ubuntu) sudo apt update && sudo apt install git -y git --version

설치 후 최초 1회 사용자 정보 설정: git config --global user.name "Your Name", git config --global user.email "you@example.com"

2) 가장 많이 쓰는 Git 명령어

저장소 시작/복제

git init
git clone <repository-url>

변경 확인

git status
git diff

스테이징/커밋

git add .
git commit -m "메시지"

브랜치

git branch
git switch -c feature/login
git switch main

원격 저장소

git remote -v
git push origin main
git pull origin main

히스토리 조회

git log --oneline --graph --decorate --all

3) 실무형 기본 작업 흐름

  1. 최신 코드 가져오기: 
    git switch main
git pull origin main
  2. 기능 브랜치 생성: 
    git switch -c feature/기능명
  3. 작업 후 변경 확인: 
    git status
git diff
  4. 스테이징과 커밋: 
    git add .
git commit -m "feat: 기능 설명"
  5. 원격 브랜치로 푸시: 
    git push -u origin feature/기능명
  6. GitHub에서 Pull Request 생성 후 코드 리뷰 및 병합

4) 자주 하는 실수와 팁

  • 커밋 메시지가 모호함: 변경 이유가 보이도록 작성하세요. 예) fix: 로그인 버튼 비활성화 조건 수정
  • main 브랜치에서 직접 작업: 기능 단위 브랜치를 습관화하면 충돌과 실수를 줄일 수 있습니다.
  • 작은 단위 커밋: 문제 발생 시 원인 파악과 롤백이 쉬워집니다.
  • 자주 동기화: git pull을 통해 원격 변경사항을 자주 가져오면 충돌을 줄일 수 있습니다.
  • 불필요 파일은 .gitignore에 등록해 저장소를 깔끔하게 관리하세요.

5) 다음 단계

이제 기본 명령어에 익숙해졌다면 rebase, stash, cherry-pick, tag를 학습해 보세요. 협업에서는 브랜치 전략(GitHub Flow, Git Flow)을 팀 규칙으로 정해두면 더욱 효율적입니다.
